================================================================================
                        QUICK START (20-30 MINUTES)
================================================================================

1. Create Supabase Project
   - Go to https://supabase.com
   - Click "New Project"
   - Fill in details
   - Wait 2-5 minutes

2. Get Your Credentials
   - Go to Settings → API
   - Copy: Project URL, anon key, service_role key
   - Go to Settings → Database
   - Copy: Connection string

3. Create .env.local
   - Run: cp .env.example .env.local
   - Edit with your credentials

4. Create Database Schema
   - Use Supabase SQL Editor
   - Paste: lib/db/migrations.sql
   - Click Run

5. Start Development
   - Run: npm run dev
   - Open: http://localhost:3000

================================================================================
                        WHAT YOU NEED FROM SUPABASE
================================================================================

When you create your Supabase project, copy these 4 values:

1. NEXT_PUBLIC_SUPABASE_URL
   - From: Settings → API → Project URL
   - Example: https://abc123.supabase.co

2. NEXT_PUBLIC_SUPABASE_ANON_KEY
   - From: Settings → API → anon public
   - Example: eyJ0eXAiOiJKV1QiLCJhbGc...

3. SUPABASE_SERVICE_ROLE_KEY
   - From: Settings → API → service_role secret
   - Example: eyJ0eXAiOiJKV1QiLCJhbGc...
   - ⚠️ KEEP THIS SECRET!

4. DATABASE_URL
   - From: Settings → Database → Connection string
   - Example: postgresql://postgres:password@db.abc123.supabase.co:5432/postgres

================================================================================
                        SECURITY REMINDERS
================================================================================

⚠️ IMPORTANT:

1. Never commit .env.local
   - It's in .gitignore for a reason
   - Contains sensitive credentials

2. Keep SUPABASE_SERVICE_ROLE_KEY secret
   - Never share it
   - Never expose to frontend
   - Never commit to git

3. Use strong passwords
   - Database password: minimum 16 characters
   - Mix of upper/lower/numbers/symbols

4. Rotate keys regularly
   - Every 90 days in production
   - Immediately if compromised

5. Use HTTPS in production
   - Always use encrypted connections
   - Never use HTTP for sensitive data
